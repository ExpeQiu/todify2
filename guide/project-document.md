# 项目文档

## 1. 项目概述

### 1.1 项目背景

Todify2 是一个旨在利用 AI 技术（特别是 Dify API）来自动化生成技术内容的平台。本项目是为了满足日益增长的高质量技术内容创作需求，旨在为开发者、技术作家和市场营销人员提供一个高效的内容创作工具。

### 1.2 项目目标

*   **核心目标**: 打造一个用户友好的工作流程页面，通过与 Dify API 集成，实现从内容搜索、技术包装、推广策略、核心稿件到演讲稿的全流程自动化生成。
*   **短期目标 (Q3 2024)**:
    *   完成用户工作流页面的前端和后端开发。
    *   实现与 Dify API 的稳定集成，支持至少三种核心内容（技术文章、推广文案、演讲稿）的生成。
    *   提供文档的在线编辑、修订和保存功能。
    *   支持将生成的文档导出为 PDF 格式。
*   **长期愿景**: 将 Todify2 发展为一个功能完善、可扩展的内容创作平台，支持更多类型的内容生成，并引入协作和版本控制功能。

## 2. 项目范围

### 2.1 核心功能

*   **用户认证**: 用户可以注册和登录系统。
*   **工作流页面**:
    *   提供一个直观的界面，引导用户完成内容生成的步骤。
    *   步骤包括：关键词搜索、技术点包装、推广策略制定、核心稿件撰写、演讲稿生成。
*   **Dify API 集成**:
    *   后端服务将调用 Dify API，根据用户输入和预设模板生成内容。
    *   支持流式和非流式响应。
*   **文档管理**:
    *   用户可以查看、编辑和保存生成的文档。
    *   所有文档与用户账户关联。
*   **导出功能**: 用户可以将最终的文档导出为 PDF 文件。

### 2.2 非功能性需求

*   **性能**: API 响应时间应在 3 秒以内（不含 Dify 生成时间），页面加载时间不超过 2 秒。
*   **安全性**: 保护用户数据和 Dify API 密钥，防止未经授权的访问。
*   **可扩展性**: 架构设计应支持未来轻松添加新的工作流和内容类型。
*   **易用性**: UI/UX 设计应简洁明了，降低用户学习成本。

## 3. 项目计划

### 3.1 技术栈

*   **前端**: React, TypeScript, Vite
*   **后端**: Node.js (Express.js), TypeScript
*   **数据库**: SQLite (用于本地开发), PostgreSQL (用于生产)
*   **ORM**: Prisma
*   **API 集成**: Dify API
