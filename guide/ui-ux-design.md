# 用户工作流程页面 - UI/UX 设计规划

## 1. 设计理念与原则

*   **简洁至上 (Simplicity First)**: 界面应干净、整洁，消除所有不必要的视觉元素，让用户能够专注于内容创作的核心任务。
*   **流程引导 (Guided Workflow)**: 设计一个清晰、线性的步骤条，让用户时刻了解自己在哪一步，以及接下来要做什么，创造一种掌控感。
*   **即时反馈 (Instant Feedback)**: 对于用户的每一个操作，系统都应提供即时的视觉反馈。在等待 AI 生成内容时，使用优雅的加载动画和进度说明，缓解用户的等待焦虑。
*   **一致性 (Consistency)**: 整个应用的色彩、字体、图标和布局风格应保持高度一致，以建立专业、可靠的品牌形象。

## 2. 页面布局与核心组件

### 2.1 总体布局

采用经典的“左侧导航 + 右侧内容区”布局。

*   **左侧导航区**: 固定一个垂直的步骤条（Stepper），清晰地展示五个核心步骤：
    1.  **智能搜索 (Search)**
    2.  **技术包装 (Package)**
    3.  **推广策略 (Promote)**
    4.  **核心稿 (Draft)**
    5.  **演讲稿 (Speech)**
    *   当前步骤高亮显示，已完成的步骤显示为“完成”状态，未开始的步骤置灰。
*   **右侧内容区**: 这是用户主要的工作区域，根据当前步骤动态显示不同的内容和交互元素。

### 2.2 核心组件设计

#### **步骤一：智能搜索 (Search)**

*   **输入框**: 一个突出、居中的搜索框，带有清晰的占位符文本，如“输入您想探索的技术关键词...”。
*   **搜索按钮**: 样式与品牌主色调一致，悬停时有动效。
*   **结果展示**: 以卡片列表的形式展示搜索结果。每张卡片包含标题、摘要和来源链接。设计应简洁，易于快速浏览。

#### **步骤二：技术包装 (Package)**

*   **编辑器**: 引入一个富文本编辑器（如 TipTap 或 TinyMCE），用于展示和编辑生成的文章初稿。
*   **配置选项**: 在编辑器上方提供一组简单的配置选项，如“文章风格”（入门/进阶）、“目标读者”等，以按钮组或下拉菜单的形式呈现。
*   **“生成”按钮**: 明确的行动号召（Call-to-Action）按钮，点击后触发 AI 生成。

#### **步骤三：推广策略 (Promote)**

*   **内容分组**: 将生成的推广文案按平台（如 Twitter, LinkedIn, 博客摘要）进行分组，使用标签页（Tabs）或可折叠面板（Accordion）进行切换。
*   **一键复制**: 每条文案旁边提供一个“复制”图标按钮，方便用户快速取用。

#### **步骤四：核心稿 (Draft)**

*   **只读预览**: 以接近最终 PDF 的样式，展示确认后的核心稿件，供用户最终审阅。
*   **“锁定并继续”按钮**: 一个醒目的确认按钮，告知用户此步骤后文章内容将不再更改。

#### **步骤五：演讲稿 (Speech)**

*   **双栏视图**: 左侧为要点提纲，右侧为详细的演讲稿内容。提纲中的每个要点都是一个锚点链接，点击可以快速跳转到右侧的相应段落。
*   **导出按钮**: 在页面右上角提供一个“导出为 PDF”的按钮，图标和文字相结合。

## 3. 色彩与字体

*   **主色调**: 采用科技蓝 (`#4A90E2`) 作为主色调，用于按钮、链接和高亮状态，传达专业和智能的品牌感。
*   **辅助色**: 使用中性灰 (`#F5F5F5` 用于背景，`#333333` 用于正文) 来保证内容的可读性。
*   **字体**: 选择一款现代、清晰的无衬线字体（如 Inter 或 Lato），保证在各种屏幕尺寸下的可读性。

## 4. 交互与动效

*   **页面过渡**: 步骤切换时，使用平滑的淡入淡出效果，避免生硬的页面跳转。
*   **加载状态**: 在调用 Dify API 时，显示一个优雅的、与品牌相关的加载动画（例如，一个跳动的文字光标或流动的脑图），并附上友好的提示语，如“AI 正在为您创作，请稍候...”。
*   **微交互**: 为按钮点击、悬停、输入框聚焦等提供细微的动效反馈，提升整体体验的精致感。


本文档旨在为 Todify2 的核心用户工作流程设计一个直观、高效且用户友好的界面。

## 1. 设计原则

- **线性引导**：用户工作流程是线性的，界面应清晰地引导用户从一步到下一步。
- **状态明确**：在任何时候，用户都应该清楚自己处于哪个步骤，以及当前步骤的状态（例如，等待输入、正在生成、已完成）。
- **即时反馈**：对于耗时操作（如 API 调用），应提供明确的加载或进度指示。
- **灵活编辑**：所有 AI 生成的内容都应是可编辑的，用户拥有最终的控制权。
- **一致性**：所有步骤的布局和交互应保持一致，降低用户的学习成本。

## 2. 总体布局

我们将采用一个**多步骤的向导式（Wizard）布局**。

- **左侧**：一个垂直的步骤导航器，显示所有工作流步骤（技术概念搜索、技术包装、推广策略等）。当前步骤会被高亮显示，已完成的步骤会有标记。
- **右侧**：主要内容区，根据当前步骤动态显示不同的内容。

```
+--------------------------------+--------------------------------------------------+
|                                |                                                  |
|  **工作流程**                  |  <当前步骤标题>                                  |
|                                |                                                  |
|  1. 技术概念搜索 (完成)        |  +----------------------------------------------+  |
|  2. 技术包装 (当前)            |  |                                              |  |
|  3. 推广策略                   |  |  输入区域 (上一阶段的输出或新输入)           |  |
|  4. 技术核心稿                 |  |                                              |  |
|  5. 演讲稿撰写                 |  +----------------------------------------------+  |
|                                |                                                  |
|                                |  [ 生成内容 ] (按钮)                             |
|                                |                                                  |
|                                |  +----------------------------------------------+  |
|                                |  |                                              |  |
|                                |  |  输出区域 (AI 生成内容，可编辑)              |  |
|                                |  |                                              |  |
|                                |  +----------------------------------------------+  |
|                                |                                                  |
|                                |  [ 保存 ] [ 导出PDF ] [ 下一步 ] (按钮组)        |
|                                |                                                  |
+--------------------------------+--------------------------------------------------+
```

## 3. 各步骤详解

### 步骤 1: 技术概念搜索

- **输入区**：一个搜索框。
- **操作**：用户输入关键词，点击“搜索”按钮。
- **输出区**：显示从 Dify API 返回的搜索结果列表。用户可以从中选择一个或多个概念作为下一步的输入。

### 步骤 2: 技术包装

- **输入区**：默认填充上一步选择的技术概念内容，用户可以进行修改。
- **操作**：点击“生成技术包装”按钮。
- **输出区**：一个富文本编辑器，显示 AI 生成的技术包装内容。用户可以自由编辑。
- **按钮组**：
    - `保存`: 将当前内容保存到数据库。
    - `导出PDF`: 将编辑器中的内容导出为 PDF 文件。
    - `下一步`: 进入“推广策略”步骤，并将当前编辑器的内容作为输入。

### 步骤 3: 推广策略

- **输入区**：默认填充上一步的技术包装内容。
- **操作**：点击“生成推广策略”按钮。
- **输出区**：富文本编辑器，显示 AI 生成的推广策略。
- **按钮组**：同上。

### 步骤 4: 技术核心稿

- **输入区**：默认填充上一步的推广策略内容。
- **操作**：点击“生成技术核心稿”按钮。
- **输出区**：富文本编辑器，显示 AI 生成的技术核心稿。
- **按钮组**：同上。

### 步骤 5: 演讲稿撰写

这是一个特殊的步骤，因为它可能需要合并多个核心稿。

- **输入区**：
    - 一个“选择技术核心稿”的按钮，点击后会弹出一个列表，显示所有已保存的技术核心稿，用户可以多选。
    - 一个文本区，显示已选核心稿的标题或摘要。
- **操作**：点击“生成演讲稿”按钮。
- **输出区**：富文本编辑器，显示 AI 生成的演讲稿。
- **按钮组**：
    - `保存`: 保存演讲稿。
    - `导出PDF`: 导出演讲稿。
    - `完成`: 结束当前工作流程。

## 4. 技术实现要点

- **前端路由**：使用 React Router 来管理不同步骤的视图。
- **状态管理**：使用 Zustand 或 React Context 来管理整个工作流的状态，包括每一步的输入和输出数据。
- **富文本编辑器**：集成一个轻量级的富文本编辑器库，如 TipTap 或 TinyMCE。
- **PDF导出**：使用 `jspdf` 和 `html2canvas` 等库来实现前端导出 PDF 的功能。
