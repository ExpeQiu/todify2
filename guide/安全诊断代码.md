# 安全的浏览器诊断方法

## ⚠️ 安全警告说明

浏览器显示"请勿将您不理解或未自行检查的代码粘贴到开发者工具控制台中"是**正常的安全警告**。

这是浏览器的安全保护机制，防止恶意代码窃取信息。

## ✅ 安全的检查方法

### 方法1：手动输入简单命令（最安全）

在控制台**手动输入**以下代码（不是粘贴，是手动输入）：

```
document.getElementById('standalone-document-editor')
```

按回车后，如果返回 `null` 表示组件不存在，如果返回一个对象表示存在。

### 方法2：逐行运行检查（推荐）

一行一行地**手动输入**，每次只输入一行：

```javascript
// 第1步：检查组件是否存在
document.getElementById('standalone-document-editor')

// 第2步：如果返回了对象，查看背景色
getComputedStyle(document.getElementById('standalone-document-editor')).backgroundColor

// 第3步：检查Markdown预览区
document.getElementById('standalone-markdown-preview')

// 第4步：查看预览区的背景色
getComputedStyle(document.getElementById('standalone-markdown-preview')).backgroundColor
```

### 方法3：使用浏览器检查工具（可视化）

1. 在开发者工具中点击 **Elements** 标签
2. 按 `Cmd+F` (Mac) 或 `Ctrl+F` (Windows) 打开搜索框
3. 搜索：`standalone-document-editor`
4. 如果能找到，说明组件存在
5. 右键点击这个元素，选择"检查"查看实际样式

## 📊 替代方法：截图诊断

如果不想运行代码，请提供以下截图：

1. **整个页面截图** - 按 `Cmd + Shift + 3` (Mac) 或 `Win + Print Screen` (Windows)
2. **开发者工具的Elements面板** - 显示DOM结构
3. **开发者工具的Console面板** - 显示任何错误信息
